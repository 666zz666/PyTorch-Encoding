Data Parallel¶

Current PyTorch DataParallel Table is not supporting mutl-gpu loss calculation, which makes the gpu memory usage very in-efficient. We address this issue here by doing CriterionDataParallel. The DataParallel compatible with SyncBN will be released later.

Modules¶

ModelDataParallel¶

class encoding.ModelDataParallel(module, device_ids=None, dim=0)[source]¶

Implements data parallelism at the module level.

This container parallelizes the application of the given module by splitting the input across the specified devices by chunking in the batch dimension. In the forward pass, the module is replicated on each device, and each replica handles a portion of the input. During the backwards pass, gradients from each replica are summed into the original module. Note that the outputs are not gathered, please use compatible CriterionDataParallel .

The batch size should be larger than the number of GPUs used. It should also be an integer multiple of the number of GPUs so that each chunk is the same size (so that each GPU processes the same number of samples).

Parameters:
  • module – module to be parallelized
  • device_ids – CUDA devices (default: all devices)

Example:

>>> net = torch.nn.ModelDataParallel(model, device_ids=[0, 1, 2])
>>> output = net(input_var)

CriterionDataParallel¶

class encoding.CriterionDataParallel(module, device_ids=None, output_device=None, dim=0)[source]¶

Calculate loss in multiple-GPUs, which balance the memory usage for Semantic Segmentation.

The targets are splitted across the specified devices by chunking in the batch dimension. Please use together with ModelDataParallel
